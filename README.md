# StlAssetManager
STLAssetManager: Веб-ориентированный программный комплекс для каталогизации и анализа 3D-моделей

### Аннотация

В данном репозитории представлено серверное решение для централизованного управления, индексации и интерактивного просмотра трехмерных ассетов в формате `.STL`. Проект реализует многопоточную архитектуру на базе Flask, автоматизирующую процессы извлечения геометрических метаданных, генерации изометрических превью-изображений и сквозного поиска по локальной библиотеке файлов.

---

### 1. Архитектура и технический стек

Программный комплекс разработан на языке Python и базируется на модульной архитектуре:
* **Ядро веб-сервера:** Фреймворк `Flask` с поддержкой потоковой обработки запросов (`threaded=True`).
* **Слой хранения данных (ORM):** `SQLAlchemy` поверх СУБД `SQLite` с настроенным журналированием в режиме `WAL` (Write-Ahead Logging) для предотвращения блокировок при конкурентном доступе.
* **Анализ геометрии:** Интеграция с библиотекой `trimesh` (модуль `core.mesh`) для низкоуровневого парсинга полигональных сеток.
* **Визуализация (Рендеринг):** Библиотека `pyvista` (на базе VTK) для аппаратного off-screen рендеринга трехмерных объектов.

---

### 2. Технические особенности реализации

#### 2.1. Потокобезопасный асинхронный сканер (`core.scanner`)
Реализован механизм фоновой индексации директорий с использованием примитивов синхронизации `threading.Lock` и сигнальных событий `threading.Event`. Процесс сканирования изолирован от основного потока веб-сервера, что исключает блокировку интерфейса (UI) и позволяет в реальном времени транслировать прогресс (`ScanProgress`) через JSON API.

#### 2.2. Дедупликация и контроль целостности данных
Идентификация файлов осуществляется не по имени, а путем вычисления криптографических хэш-сумм по алгоритму `SHA-256` блоками по 1 МБ. Это гарантирует:
* Корректную обработку коллизий и дубликатов при загрузке.
* Привязку уникального статического превью-изображения к хэшу файла.
* Автоматическое обновление метаданных в БД при изменении физического файла на диске.

#### 2.3. Механизм динамических автомиграций (`core.database`)
Внедрена легковесная система контроля версий схемы данных. При инициализации движка (`get_engine`) скрипт выполняет интроспекцию таблиц через PRAGMA-запросы к SQLite и динамически выполняет DDL-команды `ALTER TABLE ADD COLUMN`, исключая необходимость развертывания тяжелых фреймворков миграции (например, Alembic) в локальной среде.

#### 2.4. Анализ геометрической валидности
Программа дифференцирует модели по признаку "герметичности" сеток (`is_watertight`). Для замкнутых многообразий производится математический расчет объема (`volume_mm3`) и площади поверхности (`surface_mm2`), а также вычисление габаритных размеров трехмерного ограничивающего контейнера (Bounding Box) по осям X, Y, Z.

### 3. Развертывание и конфигурация

#### 3.1. Структура проекта
При первом запуске система автоматически генерирует изолированную структуру директорий в корне проекта:
* `/data/library.db` — Реляционная база данных.
* `/data/uploads/` — Локальное хранилище загруженных `.stl` файлов.
* `/static/previews/` — Кэш сгенерированных `.png` превью-изображений.

#### 3.2. Установка зависимостей
Для полноценной работы аналитического и графического ядерруют требуются следующие пакеты:
```bash
pip install Flask SQLAlchemy trimesh pyvista
```
### 3.3. Инициализация

Запуск веб-интерфейса и сетевых служб осуществляется через центральный файл:

```bash
python app.py
```
После старта сервер автоматически определяет конфигурацию сетевых интерфейсов и выводит в консоль URL-адреса для локального доступа (localhost:5000) и для подключений из внешней подсети (Wi-Fi/LAN).

### 4. Лицензия и использование
Программный комплекс поставляется "как есть" (AS IS) и предназначен для локального развертывания в целях оптимизации процессов подготовки моделей к аддитивному производству (3D-печати). Права на модификацию и распространение кода не ограничены.
